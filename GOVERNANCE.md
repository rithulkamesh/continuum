# Governance

This document describes how Continuum is maintained and how project decisions are made.

## Roles

### Maintainers

Maintainers are responsible for:

- Reviewing and merging pull requests
- Triaging issues and discussions
- Defining release scope and timelines
- Enforcing the Code of Conduct

Maintainers are expected to be active, responsive, and collaborative.

### Contributors

Contributors submit issues, documentation improvements, tests, and code changes.
All contributors are expected to follow `CODE_OF_CONDUCT.md` and `CONTRIBUTING.md`.

## Decision-Making

- Routine technical decisions are made through pull request review.
- Larger architectural changes should begin with an issue/discussion before implementation.
- When maintainers disagree, they seek consensus first; if needed, a lead maintainer makes the final call.

## Pull Request Review Policy

- At least one maintainer approval is required before merge.
- Significant behavioral or architecture changes may require two maintainer approvals.
- CI checks must pass before merge.

## Releases

- Maintainers decide release cadence based on stability and readiness.
- Release notes summarize user-visible changes.
- Security fixes may be released out of band.

## Inactive Maintainers

Maintainers who are inactive for an extended period may be moved to emeritus status to keep ownership clear.

## Changes to Governance

Changes to this document require a pull request and maintainer approval.
